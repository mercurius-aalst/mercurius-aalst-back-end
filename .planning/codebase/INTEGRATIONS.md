# External Integrations

**Analysis Date:** 2026-04-29

## APIs & External Services

**Public HTTP API:**
- ASP.NET Core Minimal API - exposes LAN tournament, player, team, sponsor, user, and auth endpoints.
  - SDK/Client: built-in ASP.NET Core request pipeline in `src/MercuriusAPI/Program.cs`.
  - Auth: JWT bearer tokens configured from `Jwt` settings in `src/MercuriusAPI/Program.cs`.
  - Endpoint groups: `auth` in `src/MercuriusAPI/Endpoints/AuthEndpoints.cs`, `lan/games` in `src/MercuriusAPI/Endpoints/GameEndpoints.cs`, `lan/matches` in `src/MercuriusAPI/Endpoints/MatchEndpoints.cs`, `lan/players` in `src/MercuriusAPI/Endpoints/PlayerEndpoints.cs`, `lan/teams` in `src/MercuriusAPI/Endpoints/TeamEndpoints.cs`, `lan/sponsors` in `src/MercuriusAPI/Endpoints/SponsorEndpoints.cs`, and `users` in `src/MercuriusAPI/Endpoints/UserEndpoints.cs`.

**Browser/API Documentation:**
- Swagger/OpenAPI - exposes API docs and interactive UI.
  - SDK/Client: `Swashbuckle.AspNetCore` and `Microsoft.AspNetCore.OpenApi` referenced in `src/MercuriusAPI/Mercurius.LAN.API.csproj`.
  - Auth: Swagger is configured for JWT bearer authentication in `src/MercuriusAPI/Extensions/JWTBuilder.cs`; secured UI customization is loaded from `src/MercuriusAPI/staticfiles/swagger-custom.js`.
  - UI setup: `app.UseSwagger()`, `app.UseSwaggerUI()`, and `app.UseSecuredSwaggerUI()` in `src/MercuriusAPI/Program.cs`.

**Image Processing:**
- Imageflow - processes uploaded images and serves optimized/static images under `/images`.
  - SDK/Client: `Imageflow.Net` and `Imageflow.Server` referenced in `src/MercuriusAPI/Mercurius.LAN.API.csproj`.
  - Auth: write access is controlled by API endpoint authorization; static `/images` serving itself is configured by middleware in `src/MercuriusAPI/Program.cs`.
  - Implementation: WebP conversion in `src/MercuriusAPI/Services/FileServices/FileService.cs`; middleware mapping in `src/MercuriusAPI/Program.cs`.

**External Outbound APIs:**
- Not detected. No `HttpClient`, SMTP, Stripe, cloud SDK, queue, or webhook client usage detected under `src/MercuriusAPI/`.

## Data Storage

**Databases:**
- PostgreSQL through Entity Framework Core.
  - Connection: `ConnectionStrings:MercuriusDB`.
  - Client: `MercuriusDBContext` in `src/MercuriusAPI/Data/MercuriusDBContext.cs`.
  - Provider: `Npgsql.EntityFrameworkCore.PostgreSQL` in `src/MercuriusAPI/Mercurius.LAN.API.csproj`.
  - Registration: `options.UseNpgsql(builder.Configuration.GetConnectionString("MercuriusDB"))` in `src/MercuriusAPI/Program.cs`.
  - Migrations: `src/MercuriusAPI/Migrations/`.
  - Startup behavior: pending migrations are applied automatically with `dbContext.Database.Migrate()` in `src/MercuriusAPI/Program.cs`.

**File Storage:**
- Local filesystem image storage.
  - Connection: `FileStorage:Location`.
  - Limits: `FileStorage:MaxFileSizeInMB`.
  - Client: `src/MercuriusAPI/Services/FileServices/FileService.cs`.
  - Validation: `src/MercuriusAPI/Services/FileServices/FileValidationService.cs`.
  - Public path: uploaded image references are returned under `images/{fileName}` by `src/MercuriusAPI/Services/FileServices/FileService.cs`.

**Caching:**
- Imageflow disk and stream caching are enabled for `/images` middleware in `src/MercuriusAPI/Program.cs`.
- No Redis, distributed cache, or external cache provider detected.

## Authentication & Identity

**Auth Provider:**
- Custom username/password authentication backed by PostgreSQL.
  - Implementation: `src/MercuriusAPI/Services/Auth/AuthService.cs`.
  - Password hashing: PBKDF2 via `Rfc2898DeriveBytes` with SHA-256 in `src/MercuriusAPI/Services/Auth/PasswordHelper.cs`.
  - Access tokens: JWTs generated by `src/MercuriusAPI/Services/Auth/Token/TokenService.cs`.
  - Refresh tokens: persisted in PostgreSQL through `RefreshTokens` on `MercuriusDBContext` in `src/MercuriusAPI/Data/MercuriusDBContext.cs`.
  - Login lockout: in-memory `LoginAttemptService` registered as a singleton in `src/MercuriusAPI/Extensions/DepedencyConfiguration.cs`.
  - Initial admin/user seed: `InitialUser` configuration consumed by `src/MercuriusAPI/Services/UserServices/UserService.cs` and triggered from `src/MercuriusAPI/Program.cs`.

**Authorization:**
- Role-based authorization uses ASP.NET Core `AuthorizeAttribute`.
  - Admin-protected groups are configured in endpoint files such as `src/MercuriusAPI/Endpoints/GameEndpoints.cs`, `src/MercuriusAPI/Endpoints/TeamEndpoints.cs`, `src/MercuriusAPI/Endpoints/SponsorEndpoints.cs`, and `src/MercuriusAPI/Endpoints/UserEndpoints.cs`.
  - Anonymous read/login endpoints use `.AllowAnonymous()` in `src/MercuriusAPI/Endpoints/AuthEndpoints.cs`, `src/MercuriusAPI/Endpoints/GameEndpoints.cs`, `src/MercuriusAPI/Endpoints/MatchEndpoints.cs`, `src/MercuriusAPI/Endpoints/SponsorEndpoints.cs`, and `src/MercuriusAPI/Endpoints/TeamEndpoints.cs`.
